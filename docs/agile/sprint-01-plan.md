# Sprint 01 - Plan de ejecucion MVP

## 1) Objetivo del sprint

Construir la base funcional y de UX para iniciar el producto sin bloquear iteraciones futuras.

## 2) Duracion sugerida

- 2 semanas.
- Fecha inicio sugerida: 2026-06-17.
- Fecha fin sugerida: 2026-07-01.

## 3) Alcance de Sprint 01

Issues incluidas:

- #7 - [UX] Flujo Figma MVP alineado con SDD
- #1 - [EP-01] Configuracion de evento y mesas
- #2 - [EP-02] Captura de datos de invitados

## 4) Orden de ejecucion recomendado

1. #7 (Figma) para definir flujos y wireframes de admin/invitado.
2. #1 (evento y mesas) para habilitar la base del dominio.
3. #2 (captura de invitados) para completar la entrada principal de datos.

## 5) Dependencias

- #1 depende de decisiones de flujo en #7.
- #2 depende parcialmente de #1 y #7.

## 6) Entregables esperados

- Flujos UX validados en Figma para historias del sprint.
- Configuracion operativa de evento/mesas.
- Formulario de invitado con datos sociales y necesidades especiales.
- Actualizacion de docs si hay cambios de alcance o criterio.

## 7) Criterio de cierre del sprint

El sprint se considera cerrado cuando:

- Todas las issues del sprint estan en estado cerrado.
- Cada issue cumple sus criterios de aceptacion.
- SDD-01 sigue alineado con el comportamiento implementado.
- Existe evidencia minima de pruebas funcionales basicas.
- Quedan registrados aprendizajes para Sprint 02.

## 8) Riesgos del sprint y mitigacion

- Riesgo: sobrecargar #2 con demasiados campos.
  - Mitigacion: mantener MVP, mover extras a backlog.
- Riesgo: decisiones UX tardias.
  - Mitigacion: cerrar wireframes core al inicio del sprint.
- Riesgo: reglas ambiguas en formulario.
  - Mitigacion: validar textos y restricciones en SDD-01.

## 9) Definition of Done del Sprint 01

- Historias cerradas con criterios de aceptacion verificados.
- Documentacion actualizada (SDD/ADR si aplica).
- Backlog de Sprint 02 preparado con prioridades claras.

## 10) Comentarios para principiantes

- **Sprint:** bloque corto de trabajo con objetivo claro.
- **Alcance del sprint:** lista de trabajo comprometida para ese bloque.
- **Criterio de cierre:** regla objetiva para decidir si el sprint termino bien.
