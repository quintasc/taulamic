# Agile para Principiantes (aplicado a este proyecto)

## 1) Idea principal

Agile no significa "trabajar sin plan".  
Significa planificar en ciclos cortos, entregar valor pronto y mejorar con feedback.

## 2) Ciclo simple recomendado

1. Definir objetivo de ciclo (ejemplo: cerrar SDD-01).
2. Seleccionar pocas tareas importantes.
3. Ejecutar y validar.
4. Revisar resultado.
5. Ajustar siguiente ciclo.

## 3) Ejemplo de backlog inicial

- Crear SDD-00 y SDD-01.
- Definir modelo de datos base.
- Definir reglas duras y blandas del motor.
- Prototipo UI para admin e invitado.
- Flujo de aprobacion y publicacion.

## 4) Buenas practicas para principiantes

- Mantener tareas pequenas y claras.
- Evitar empezar muchas cosas a la vez.
- Documentar decisiones justo cuando se toman.
- Validar pronto con usuarios reales (novios, planners o salon).

## 5) Comentarios para principiantes

### "Iterar"

Hacer una version, aprender, mejorar y repetir.

### "Feedback"

Opinion util de personas usuarias para corregir rumbo.

### "Priorizar"

Elegir que hacer primero segun valor y riesgo.
