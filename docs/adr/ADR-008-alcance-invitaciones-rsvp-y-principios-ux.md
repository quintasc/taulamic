# ADR-008 - Alcance de invitaciones/RSVP y principios UX

- Estado: Aceptado
- Fecha: 2026-06-17

## Contexto

El producto nacio como solucion de distribucion de mesas.
Para crear una experiencia completa de gestion de invitados, el organizador tambien necesita:

- enviar invitaciones,
- confirmar asistencia (RSVP),
- y operar en una interfaz clara para usuarios no tecnicos.

## Decision

El MVP ampliado incluye:

1. Envio de invitaciones (individual y masivo).
2. Gestion de RSVP (`si`, `no`, `pendiente`).
3. Direccion UX/UI sobria, elegante y de baja friccion.

## Benchmark de referencia

Se usaran como inspiracion funcional y de usabilidad:

- PerfectTablePlan
- Planning Pod
- Prismm

Nota:

- benchmark solo como referencia de buenas practicas, sin copia literal.

## Motivos de la decision

- Aumenta valor del producto para organizadores.
- Reduce herramientas externas necesarias.
- Mejora calidad de datos para el motor de distribucion.
- Refuerza adopcion por simplicidad de uso.

## Consecuencias positivas

- Plataforma mas completa desde fases tempranas.
- Mayor conversion de invitados al proceso digital.
- Mejor trazabilidad del ciclo completo del evento.

## Consecuencias negativas

- Aumenta alcance funcional del MVP.
- Exige mayor disciplina de UX para evitar complejidad.

## Condiciones para revisar esta decision

Reevaluar cuando:

- el alcance comprometa tiempos criticos de lanzamiento,
- o los flujos de invitacion/RSVP requieran integraciones externas complejas.

## Comentarios para principiantes

- **RSVP:** respuesta de asistencia de un invitado.
- **Benchmark:** comparar con otras soluciones para aprender buenas practicas.
- **Baja friccion:** que la app pida el menor esfuerzo posible al usuario.
